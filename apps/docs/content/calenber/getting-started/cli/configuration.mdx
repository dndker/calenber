## 옵션

### path

`path`는 생성되는 컴포넌트의 기본 경로를 설정해요.

calenber이 필요로 하는 폴더나 파일들은 `path`로 지정된 경로의 하위에 생겨요.
`calenber`의 root 폴더는 유저가 입력할 수 있게 하되, 그 내부에 생성되는 폴더나 파일들은 고정되는 형식이에요.

```json title="calenber.json"
{
    "path": "./calenber"
}
```

만약 위와 같이 설정했다면, `./calenber` 폴더가 생성이 되고,
그 안에 `ui`, `utils`, `hooks`와 같은 폴더들이 생기게 될거에요.

### rsc

[React Server Components](https://react.dev/reference/rsc/server-components)를 사용할지 설정해요.

`true`로 설정하면 컴포넌트에 `use client` directive가 추가돼요.

```json title="calenber.json"
{
  "rsc": true | false // default: false
}
```

### tsx

TypeScript를 사용할지 설정해요.

`true`로 설정하면 `.tsx` 컴포넌트가, `false`로 설정하면 `.jsx` 컴포넌트가 생성돼요.

```json title="calenber.json"
{
  "tsx": true | false // default: true
}
```

### telemetry

텔레메트리 데이터 수집 여부를 설정해요.

ST Design CLI는 개선을 위해 익명 사용 데이터를 수집해요.
개인 정보는 수집하지 않으며, 커맨드 사용 패턴과 성능 지표만 수집해요.

```json title="calenber.json"
{
  "telemetry": true | false // default: true
}
```

#### 수집되는 데이터

- 사용된 커맨드 (init, add, add-all)
- 설정 옵션 (tsx, rsc 사용 여부)
- 성능 지표 (실행 시간)
- 기술 환경 (Node.js 버전, OS 플랫폼)

#### 수집되지 않는 데이터

- 프로젝트 경로, 파일 이름
- 컴포넌트 이름
- 소스 코드
- 개인 식별 정보

#### 비활성화 방법

**1. calenber.json 설정**

```json
{
    "telemetry": false
}
```

**2. 환경 변수 설정**

```bash
export DISABLE_TELEMETRY=true
# 또는
export SEED_DISABLE_TELEMETRY=true
```
